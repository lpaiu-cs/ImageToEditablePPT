# [Historical / Obsolete] VLM + Local CV 하이브리드 아키텍처 전면 개편 지시서

> Status: historical / obsolete v2-era instruction
>
> Current source of truth: `plan.md`

현재의 Bottom-Up 방식(픽셀 -> 선 -> 박스 조립)은 그라데이션, 다크 테마, 글로우(Glow) 효과가 포함된 모던 인포그래픽에서 완전히 실패했음이 입증되었다. 기존의 `Global Segment Graph`나 전역 `Canny/Mean-shift` 의존성을 대폭 축소하고, 의미(Semantic)를 먼저 추출한 뒤 좌표를 맞추는 **Top-Down 하이브리드 파이프라인**으로 코드를 전면 재설계하라. 

다음 4단계의 모듈 신설 및 개편을 즉시 실행하라.

### 단계 1: VLM 프록시 모듈 신설 (`vlm_parser.py`)
전체 이미지의 논리적 구조를 단번에 파악하기 위해 VLM(Vision-Language Model, 예: OpenAI GPT-4o, Claude 3.5 Sonnet, 또는 Gemini 호환 API)을 호출하는 모듈을 새로 만든다.

* **구현 목표:** 이미지를 Base64로 인코딩하여 VLM API에 전송하고, 정형화된 JSON 형태의 다이어그램 구조(IR: Intermediate Representation)를 반환받는다.
* **JSON Schema 요구사항:**
  ```json
  {
    "nodes": [
      {"id": "n1", "type": "box|cylinder|document|text_only", "text": "Vector Store", "approx_bbox": [x_min, y_min, x_max, y_max]}
    ],
    "edges": [
      {"source": "n1", "target": "n2", "type": "solid_arrow|dashed_arrow|line", "label": "retrieves"}
    ]
  }
  ```
* **작업 지시:** `src/image_to_editable_ppt/vlm_parser.py`를 신설하고, VLM 호출 함수 및 JSON 파싱/검증 로직(Pydantic 권장)을 구현하라. (API Key는 환경 변수 `VLM_API_KEY`를 사용하도록 처리할 것).

### 단계 2: `detector.py` 및 `fitter.py`의 역할 변경 (Global -> Local CV)
기존에는 전체 이미지에서 노이즈와 싸우며 윤곽선을 찾았지만, 이제는 VLM이 제안한 `approx_bbox` 내부만 국소적(Local)으로 검사한다.

* **구현 목표:** VLM의 바운딩 박스는 약간 부정확할 수 있으므로, 해당 영역을 자른(Crop) 뒤 OpenCV를 사용해 실제 픽셀 경계(모서리 좌표, 둥근 모서리 곡률 등)에 정확히 스냅(Snap)시킨다.
* **작업 지시 (`detector.py` 개편):**
  1. 전역(Global) Hough Line 탐색과 복잡한 `non_diagram_mask` 로직을 비활성화(혹은 레거시 Fallback으로 이동)하라.
  2. VLM에서 전달받은 각 노드의 `approx_bbox` 영역에 대해서만 여유 공간(Padding)을 두고 Crop하라.
  3. Crop된 작은 이미지 내에서만 `cv2.findContours` 또는 극단적인 대비 조정(Contrast Thresholding)을 수행하여 **가장 바깥쪽 외곽선(Outer Edge)**을 정확히 찾아 `exact_bbox`로 보정하라.
  4. 텍스처나 아이콘 때문에 외곽선이 무너진 경우, `approx_bbox` 자체를 `exact_bbox`로 확정(Fallback)하는 안전장치를 두어라.

### 단계 3: `repair.py`를 `router.py` (위상 기반 라우팅)로 전면 개체
기존처럼 끊어진 선분을 픽셀 단위로 이어붙이는 방식을 폐기한다. VLM이 엣지(Edge)의 존재를 이미 알려주었으므로, 프로그래밍 방식으로 선을 그린다.

* **구현 목표:** 의미론적으로 연결된 두 노드 간의 PPTX 연결선(Connector) 경로를 생성한다.
* **작업 지시 (`router.py` 신설 또는 `repair.py` 개편):**
  1. VLM의 `edges` 데이터를 순회하며 `source` 노드와 `target` 노드의 보정된 `exact_bbox`를 가져온다.
  2. 두 박스의 중심점 좌표를 기준으로 상/하/좌/우 중 가장 가까운 앵커(Anchor) 포인트를 계산하라.
  3. 직교 라우팅(Orthogonal Routing, 맨해튼 거리 기반) 알고리즘을 구현하여, 두 앵커를 잇는 꺾인 선(Elbow Connector)의 제어점(Control Points) 좌표를 산출하라.
  4. 픽셀 이미지 상에 실제 선분이 끊겨있든, 텍스트에 가려져 있든 상관하지 마라. VLM이 연결을 지시했다면 강제로 생성(Force Route)하라.

### 단계 4: 파이프라인 흐름 재구성 (`pipeline.py`)
위의 모듈들을 하나로 엮어 새로운 실행 경로를 확립한다.

* **작업 지시 (`pipeline.py` 수정):**
  1. `run_pipeline(image_path)`의 시작점에 `vlm_parser.extract_structure()`를 배치한다.
  2. 추출된 VLM 구조를 순회하며 `detector.refine_node_geometry()`를 호출해 기하학적 정밀도를 높인다.
  3. `router.generate_connections()`를 호출해 노드 간의 엣지를 생성한다.
  4. 정제된 Node와 Edge 리스트를 기존의 `exporter.py` (PPTX 생성기) 포맷에 맞게 변환(Translate)하여 전달한다.

**[주의 사항]** 이 개편의 핵심은 **"픽셀이 아니라 문맥(Context)을 먼저 믿는 것"**이다. 그라데이션이나 발광 효과(Glow)로 인해 OpenCV가 모서리 검출에 실패하더라도, 해당 객체를 절대 버리지(Drop) 말고 VLM이 제안한 초기 거친 박스(Coarse Box)를 사용해 무조건 PPTX 요소로 살려내야 한다.
